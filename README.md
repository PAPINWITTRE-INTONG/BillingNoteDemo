# Billing Note Generator

เว็บแอปสำหรับสร้าง **ใบวางบิล (Billing Note)** จากไฟล์ Excel ที่ export ออกมาจากระบบบัญชี/ERP (เช่น Odoo) โดยอัตโนมัติ — จัดกลุ่มรายการตามลูกค้าและตามเลขที่ใบวางบิล (หรือตามเดือน), คำนวณยอดรวม, แปลงยอดเป็นตัวหนังสือภาษาไทย, จัดหน้าเอกสารให้พอดี A4 (หรือขนาดกระดาษอื่น) แล้ว export ออกเป็น PDF / Excel ให้พร้อมพิมพ์หรือนำเข้าระบบต่อได้ทันที

ทำงานทั้งหมดในเบราว์เซอร์ (client-side) ไม่มีการส่งข้อมูลขึ้นเซิร์ฟเวอร์ใดๆ

## วิธีใช้งาน

1. เปิดเว็บแอป แล้วลากไฟล์ Excel ต้นทาง (`.xlsx`) วางในกล่องอัปโหลด หรือกดเพื่อเลือกไฟล์
2. แอปจะอ่านไฟล์ จัดกลุ่มรายการเป็นรายลูกค้าอัตโนมัติ แสดงรายชื่อลูกค้าทั้งหมดในแถบด้านซ้าย
3. เลือกลูกค้าจากแถบด้านซ้าย (หรือค้นหาชื่อบริษัทได้) เพื่อดู/แก้ไขเอกสารของลูกค้านั้น
4. ช่องที่แก้ไขได้ในตัวเอกสาร (ถ้าข้อมูลจากไฟล์ต้นทางไม่มีมาให้ครบ): เลขที่ใบวางบิล (BL), วันที่เอกสาร, วันครบกำหนด, ที่อยู่ลูกค้า, ชื่อผู้ขาย
5. เลือกขนาดกระดาษที่ต้องการจากดรอปดาวน์ (ค่าเริ่มต้น A4) — รองรับ A2–A6, B4/B5, Letter/Legal/Tabloid/Executive, Folio หรือกำหนดขนาดเอง
6. Export:
   - **Export ใบนี้** — ส่งออกเฉพาะลูกค้าที่กำลังเปิดดูอยู่ เลือกได้ว่าจะเอา PDF, Excel หรือทั้งคู่
   - **Export ทั้งหมด (.zip)** — ส่งออกทุกลูกค้าในไฟล์เดียวเป็น .zip (มี PDF + Excel ของทุกคน พร้อมไฟล์สำหรับนำเข้า Odoo)
7. กด ⚙ (ตั้งค่า) ที่มุมบนซ้ายเพื่อแก้ไขข้อมูลบริษัทผู้ออกเอกสาร (ชื่อ/ที่อยู่/เลขผู้เสียภาษี/บัญชีธนาคาร ฯลฯ) ที่จะแสดงในทุกใบวางบิล

จุด **●** หน้าชื่อลูกค้าในแถบด้านซ้าย หมายถึงลูกค้ารายนั้นยังกรอกเลขที่เอกสาร/วันที่เอกสารไม่ครบ

## รูปแบบไฟล์ Excel ที่ต้องใช้

ไฟล์ต้องเป็น `.xlsx` (แถวแรกเป็นหัวคอลัมน์) รองรับ 2 รูปแบบ โดยแอปจะเลือกอัตโนมัติจากการเช็คว่ามีคอลัมน์ `Billing Number` หรือไม่:

### รูปแบบที่ 1 — มีเลขที่ใบวางบิลอยู่แล้ว (แนะนำ)

ใช้เมื่อไฟล์ export มาจากระบบที่มีเลขที่ใบวางบิล (BL) และวันที่เอกสารอยู่แล้ว แอปจะจัดกลุ่มตาม **ลูกค้า → เลขที่ใบวางบิล** โดยแต่ละเลขที่ BL จะกลายเป็นเอกสาร/หน้าของตัวเอง กรอกข้อมูล BL Number, วันที่เอกสาร, วันครบกำหนด, ที่อยู่ ให้ครบอัตโนมัติ

คอลัมน์ที่ต้องมี:

| คอลัมน์ | คำอธิบาย |
|---|---|
| `Billing Number` | เลขที่ใบวางบิล (ใช้จัดกลุ่มรายการเป็น 1 เอกสาร) |
| `Partner/Name` | ชื่อลูกค้า |
| `Billing Date` | วันที่ออกเอกสาร |
| `Billing Due date` | วันครบกำหนดชำระของเอกสาร |
| `Invoice Address/Address` | ที่อยู่สำหรับวางบิล |
| `Number` | เลขที่ใบแจ้งหนี้ (invoice) ต่อรายการ |
| `Invoice/Bill Date` | วันที่ของใบแจ้งหนี้แต่ละรายการ |
| `Due Date` | วันครบกำหนดของใบแจ้งหนี้แต่ละรายการ |
| `Total Signed` | ยอดเงินของรายการนั้น |
| `Sale Tags` (ไม่บังคับ) | ใช้ดึงชื่อผู้ขาย (seller) อัตโนมัติ รูปแบบ `ชื่อผู้ขาย - ...` |

### รูปแบบที่ 2 — ยังไม่มีเลขที่ใบวางบิล

ใช้เมื่อไฟล์ต้นทางมีแค่รายการใบแจ้งหนี้ ไม่มีเลขที่ใบวางบิล แอปจะจัดกลุ่มตาม **ลูกค้า → เดือน** ของแต่ละรายการแทน (เดือนล่าสุดขึ้นก่อน) แล้วเว้นช่อง BL Number/วันที่เอกสาร/ที่อยู่ ให้กรอกเองในหน้าเว็บ

คอลัมน์ที่ต้องมี:

| คอลัมน์ | คำอธิบาย |
|---|---|
| `Partner` | ชื่อลูกค้า |
| `Number` | เลขที่ใบแจ้งหนี้ |
| `Date` | วันที่ของใบแจ้งหนี้ (ใช้จัดกลุ่มเป็นรายเดือนด้วย) |
| `Due Date` | วันครบกำหนดของใบแจ้งหนี้ |
| `Total in Currency Signed` หรือ `Total Signed` | ยอดเงินของรายการนั้น |
| `Sale Tags` (ไม่บังคับ) | ใช้ดึงชื่อผู้ขายอัตโนมัติ |

> หมายเหตุ: ชื่อคอลัมน์ตรวจสอบแบบไม่สนตัวพิมพ์เล็ก/ใหญ่บางส่วน แต่ควรใช้ชื่อคอลัมน์ตามตารางด้านบนให้ตรงที่สุดเพื่อไม่ให้ข้อมูลตกหล่น

## ไฟล์ที่ export ออกมา

- **PDF** — ใบวางบิลพร้อมพิมพ์ จัดหน้าอัตโนมัติ แต่ละ BL/เดือนจะปิดท้ายด้วยยอดรวม + ช่องลงนามของตัวเอง จะขึ้นหลายหน้าก็ต่อเมื่อรายการในเดือน/BL นั้นเยอะเกินจะอยู่หน้าเดียว
- **Excel** — ใบวางบิลรูปแบบเดียวกับ PDF แต่เป็นไฟล์ Excel (1 ชีตต่อ 1 BL)
- **ไฟล์สำหรับนำเข้า Odoo** (รวมอยู่ใน Export ทั้งหมด .zip) — ไฟล์ Excel สรุปเลขที่ BL/วันที่/ลูกค้า/รายการใบแจ้งหนี้ ในรูปแบบที่พร้อมนำเข้ากลับเข้า Odoo

## โครงสร้างโปรเจกต์

```
public/
  index.html   — หน้าเว็บหลัก (โครงสร้าง UI)
  styles.css   — สไตล์
  app.js       — ตรรกะทั้งหมด (อ่านไฟล์, จัดกลุ่ม, จัดหน้า, export PDF/Excel)
worker.js      — Cloudflare Worker entry point (serve static assets)
wrangler.toml  — การตั้งค่า deploy บน Cloudflare Workers
```

## Deploy

โปรเจกต์นี้ deploy เป็น Cloudflare Worker (static assets) ผ่าน `npx wrangler deploy` โดยเชื่อมกับ GitHub repo นี้แล้ว push ขึ้น branch `main` จะ trigger build อัตโนมัติ
